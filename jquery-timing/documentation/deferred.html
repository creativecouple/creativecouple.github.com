Timed Invocation Chain (tic) vs. jQuery.Deferred

jQuery also has another mechanism to handle deferrals – the jQuery.Deferred object:

“jQuery.Deferred, introduced in version 1.5, is a chainable utility object that can register multiple callbacks into callback queues, invoke callback queues, and relay the success or failure state of any synchronous or asynchronous function.”

That means jQuery.Deferred works well with callback functions. In contrast the tic concept wants to remove as much callback functions out of your code as possible. So the timed invocation chain tries to achieve similar things like jQuery.Deferred, but it takes a complete different way to reach that goal.

Instant invocation

The Deferred concept in jQuery allows to add callback functions even if the deferred object already resolved. In that case the callback method is invoked instantly. Take a look at the following example:

{% highlight javascript %} var def = new $.Deferred(); // create a Deferred object def.done(myCallbackFunction); // register success callback //… def.resolve(); // succeed the Deferred // → myCallbackFunction() has been called def.done(myOtherCallback); // register another success callback // → myOtherCallback() has been called immediately {% endhighlight %}

As you will see below the tic concept also provides the possibility to add later actions after its delays timed out, its waiting events triggered, or even inside of sequential and parallel loops.
